Quem configura o quê
Etapa Responsável Pode alterar depois? Impacto de mudança
Cadastro do Cliente Admin TCM Sim Baixo — dados cadastrais apenas
Estrutura de Unidades Admin TCM Sim — adicionar filiais Médio — nova filial requer operações e usuários
Cadastro de Operações Admin TCM Sim Alto — afeta produtos e eventos vinculados
Seleção de Produtos Admin TCM Sim — contratual Alto — habilita ou remove painéis para todos os usuários daquela operação
Seleção de Eventos Gestor do cliente Sim — configuração viva Imediato — altera o que os operadores veem na fila
Cadastro de Usuários Gestor do cliente Sim — autogestão Baixo — isolado por usuário
Configuração viva
🔄
O que pode ser alterado sem reconfigurar tudo
Após o onboarding inicial, o sistema deve permitir evolução independente de cada camada — sem travar o cliente em um setup fixo.
Adicionar uma Filial nova sem mexer nas operações e usuários existentes
Habilitar um produto novo numa operação — todos os usuários daquela operação passam a ter acesso automaticamente
Ajustar thresholds de eventos sem reenvolver o Admin TCM — autonomia do Gestor do cliente
Criar ou desativar usuários sem impactar a configuração da operação
Vincular um usuário a uma nova operação sem alterar as demais vinculações
Princípio de design: mudanças em camadas superiores (Cliente, Unidade, Operação) impactam para baixo — mas mudanças em camadas inferiores (Usuário, Eventos) são isoladas e não afetam o restante da hierarquia.
